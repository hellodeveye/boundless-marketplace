# Q&A Hub 工作流

1. 读取 `apps/feedback-hub/skills/feedback-hub/*.md` 下的分类文件（忽略 `SKILL.md`）。
2. 每个文件代表一个主题（例如 `login-ux.md`、`service-experience.md`）。
3. 先做问题匹配：优先精确匹配 `Q:`；若未命中，再做关键词匹配（忽略标点和语气词）。
4. 命中后直接返回对应 `A:`，不扩展背景分析。
5. 新内容优先以 `Q:`/`A:` 形式追加到已有主题文件；若不匹配再新增主题文件。

# 编写规则

1. 一类问题一个 Markdown 文件。
2. 在一个主题文件内使用多个问答对。
3. 问答统一格式：`## Q: ...` 下一行写 `A: ...`。
4. 不使用 YAML frontmatter。
5. 优先使用简洁中文，避免模糊术语。

# 分类文件模板

```md
# <分类名称>

## 范围
- 说明该分类覆盖哪些问题。
- 保持分类边界稳定，避免频繁改名。

## Q: <用户常见问题>
A: <简洁明确的回答>

## Q: <另一条问题>
A: <另一条回答>
```

# 回答要求

1. 若知识库中有明确答案：默认只输出 1-2 句，不加前置解释，不加客套，不反问。
2. 除非用户明确要求“展开说明/详细步骤”，否则不要输出分点长文。
3. 命中答案时，禁止输出“当前知识库暂无该问答”。
4. 未命中答案时，仅输出一句：`当前知识库暂无该问答（建议补充到 <file>.md）。`
5. 默认不要追加“需要我帮你添加吗？”这类反问，除非用户明确要求编辑知识库。
6. 禁止输出检索过程旁白，例如“我来查看”“让我搜索”“现在查看目录”等。
7. 禁止使用知识库外的泛化脑补，例如“根据一般经验”“常见做法是”。
8. 命中时优先复用原始 `A:` 文本，最多做轻微压缩，不新增未出现的数字或规则。
9. 命中时必须逐字复用 `A:`（除去开头 `A:` 标签），不得改写、扩写、解释。
10. 默认输出为单行纯文本；除非用户明确要求来源，否则不附加文件路径。

# 默认输出模板

1. 命中：`<直接答案>` 或 `<直接答案>（来源：<file>.md）`
2. 未命中：`当前知识库暂无该问答（建议补充到 <file>.md）。`

# 严格回答模式（优先级最高）

1. 若问题命中现有 `Q:`，最终输出必须等于该条 `A:` 的原文。
2. 不得添加任何额外段落、标题、建议、免责声明、过程说明。
3. 若存在多条近似 `Q:`，选择最接近的一条；仍不确定则走未命中模板。

# 禁止输出清单

1. 过程叙述：`我来查看...`、`让我搜索...`、`现在我来...`
2. 常识兜底：`根据一般经验...`、`通常系统会...`（当知识库已命中时）
3. 冗余建议：`您希望我帮您添加到知识库吗？`（用户未明确要求编辑时）

# 示例

1. 用户问：`连续输错密码会发生什么？`
2. 正确输出：`连续多次失败会触发短时保护，用户可稍后再试或通过“找回密码”流程恢复登录。`
